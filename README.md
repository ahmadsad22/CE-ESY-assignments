# تنفيذ المخزن الدائري (Circular Buffer Implementation)


## مفهوم المخزن الدائري
المخزن الدائري هو مصفوفة ثابتة الحجم يتم التعامل معها كأنها حلقة وصل مستمرة. يتميز عن المخزن الخطي بأنه يعيد استخدام الأماكن التي تم قراءتها وتفريغها تلقائياً.



## محتويات المشروع
يتضمن الملف المصدري `main.c` العمليات التالية:
1. **التهيئة (Initialization):** تصفير المؤشرات وتجهيز المخزن.
2. **الكتابة (Write):** إضافة محارف إلى المخزن مع التحقق من حالة الامتلاء (Overflow).
3. **القراءة (Read):** استخراج المحارف من المخزن مع التحقق من حالة الفراغ (Underflow).
4. **الدالة الرئيسية (Main):** تطبيق عملي يطلب اسم المستخدم، يضيف له اللاحقة "ESY-CE"، يخزنه، ثم يعرضه.

## كيفية الاختبار (Testing)
للتأكد من صحة معالجة الأخطاء البرمجية في حالات الامتلاء:
* **لاختبار حالة الامتلاء (Overflow):** قمت بتغيير قيمة الثابت `SIZE` في السطر رقم 8 من الكود إلى قيمة صغيرة (مثل **10**) ثم أعدت تشغيل البرنامج. ستلاحظ ظهور رسائل الخطأ عند محاولة تخزين الاسم كاملاً.
* **الحالة الطبيعية:** القيمة الافتراضية هي **50** لضمان استيعاب الأسماء الطويلة مع اللاحقة.

## كيف يعمل الحل؟
- يستخدم المشروع مؤشرين: `head` (للكتابة) و `tail` (للقراءة).
- يتم حساب الموقع التالي باستخدام عملية باقي القسمة `%` لضمان الدوران داخل حدود المصفوفة.
